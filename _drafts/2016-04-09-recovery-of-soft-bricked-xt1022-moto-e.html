My mom owns old Motorola E-2014 edition (condor). This was a great device at that time in the entry segment.
The only problem with this device is that it comes with extremely limited internal memory. Merely 4 GB. This limit was causing a lot of trouble recently, especially after upgrading to lollipop update.

Recently CM started providing nightly CM13 builds for condor. CM13 is based on Android Marshmallow. One of the great features of Marshmallow is that you can format external sd card to be treated as internal sd card, where apps can be installed. This is a cheaper way of getting extra memory for devices with limited internal storage.

So, next task was to install CM-13 nightly build on condor (There are no RC/Snapshot/release builds available as of 20160409).

Steps followed:
  • Install Android Platform Tools.
    • I primarily work on ubuntu, if windows is your preferred choice you need to install appropriate USB drivers from Motorola.
    • This would install fastboot and adb on your device.
    • Make sure that your device is detected by adb, by using adb devices command.
  • Fastboot
    • Start your phone by pressing power and 'volume down' key for few seconds.
    • If the hardware shorcut is not working, then you can also connect your phone to adb and run `adb reboot fastboot`
    • Once in fastboot mode, next step is to flash custom recovery
